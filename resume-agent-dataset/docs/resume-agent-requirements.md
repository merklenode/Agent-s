# Resume Building Requirements

Updated: 2026-06-23

This dataset is filtered from the MNC and startup target research. It removes company names, links, and rankings, and keeps only reusable resume requirements.

## Developing Profiles

### Software Engineer Core

- Target resume identity: entry-level software engineer with strong CS fundamentals and project proof.
- Required knowledge: data structures and algorithms, clean coding, debugging, testing basics, REST APIs, databases, Git, problem solving.
- Required skills: TypeScript, Python, backend APIs, frontend basics, SQL/PostgreSQL, project documentation.
- Resume proof needed: 2-3 strong engineering projects with clear ownership, measurable facts where true, GitHub links, and concise technical bullets.
- Special notes: this is the base profile for both MNC and startup applications.

### Backend / Platform Engineer

- Target resume identity: backend engineer who can build reliable APIs, data workflows, and platform services.
- Required knowledge: REST APIs, database design, SQL, authentication basics, reliability, scalability, system design basics.
- Required skills: Python, TypeScript, FastAPI or similar backend framework, PostgreSQL, Prisma/ORM, Docker basics, validation, testing.
- Resume proof needed: backend project architecture, API endpoints, database schema, performance or reliability notes, testing details.
- Special notes: useful for MNC backend roles, fintech, SaaS, cloud, and platform teams.

### AI Automation / AI Product Engineer

- Target resume identity: engineer who can build AI tooling, agent workflows, and product features around LLM APIs.
- Required knowledge: LLM API usage, agent workflows, prompt/tool design, evaluation basics, safety-aware implementation, automation pipelines.
- Required skills: OpenAI API or similar model APIs, TypeScript, Python, workflow orchestration, GitHub automation, database-backed AI tools.
- Resume proof needed: Agent-s demo, AI project case study, screenshots, architecture notes, what was automated, measurable time saved if true.
- Special notes: very important for AI startups, developer platform teams, and AI product engineering roles.

### Developer Tools / Developer Experience Engineer

- Target resume identity: builder of tools that improve developer workflows.
- Required knowledge: CLI or web tooling, APIs, documentation, GitHub workflows, editor/dev workflow awareness, developer ergonomics.
- Required skills: TypeScript, Node.js, GitHub API/Octokit, MCP-style workflows, automation scripts, clean docs.
- Resume proof needed: tool demo, workflow before/after, GitHub automation report, documentation link, real usage example.
- Special notes: strong fit for devtools startups, AI coding tools, platform teams, and open-source-oriented roles.

### Cloud / Infrastructure Engineer

- Target resume identity: software engineer with cloud, deployment, infrastructure, and reliability awareness.
- Required knowledge: cloud basics, deployment flow, containers, environment config, logs, monitoring, reliability, distributed systems basics.
- Required skills: Docker, backend deployment, PostgreSQL hosting, API reliability, cloud concepts for AWS/Azure/GCP.
- Resume proof needed: deployed project link, deployment notes, environment architecture, logging/monitoring details.
- Special notes: cloud keywords should be honest; add proof before claiming deep cloud experience.

### Frontend / Full-stack Product Engineer

- Target resume identity: engineer who can ship user-facing product features across frontend and backend.
- Required knowledge: React/Next.js, UI state, API integration, auth flows, frontend performance, product quality.
- Required skills: React, Next.js, TypeScript, API integration, responsive UI, forms, validation, accessibility basics.
- Resume proof needed: live portfolio, screenshots, product UI examples, clear frontend/backend ownership notes.
- Special notes: startups value visible shipped work; MNCs still need clean engineering framing.

### Systems / Performance Engineer

- Target resume identity: engineer with low-level, performance, blockchain, or infrastructure depth.
- Required knowledge: C/C++, OS/concurrency basics, networking, performance, distributed systems, cryptographic/hash concepts.
- Required skills: C++, CMake, WebAssembly, Solidity/Hardhat where relevant, IPFS, Merkle roots, performance reasoning.
- Resume proof needed: systems project proof, benchmark/performance notes, architecture explanation, code-quality evidence.
- Special notes: useful for systems, blockchain, embedded, infrastructure, and performance-heavy roles.

### Data / Search / Database Engineer

- Target resume identity: engineer with strong data handling, SQL, search, and database-backed application experience.
- Required knowledge: SQL, schema design, indexing basics, data integrity, search/retrieval concepts, vector database awareness.
- Required skills: PostgreSQL, data processing, backend APIs, PDF/Excel workflows, retrieval/search project basics.
- Resume proof needed: SQL project evidence, schema examples, data pipeline notes, search/vector demo if available.
- Special notes: add this profile if targeting database, backend, fintech, AI retrieval, or data engineering roles.

## Resume Sections To Build

### Header

- Required data: name, email, phone, location, GitHub, LinkedIn, portfolio.
- Missing or check: clean links, consistent names, professional filenames.

### Summary

- Required data: 2-3 lines describing the strongest target profile.
- Best pattern: software engineer with backend, AI automation, and full-stack project proof.
- Avoid: generic claims without project evidence.

### Technical Skills

- Required categories: Languages, Backend, Frontend, Databases, AI/Automation, Cloud/DevOps, Tools, Systems.
- Strong keywords to include only if true: TypeScript, Python, React, Next.js, FastAPI, PostgreSQL, Prisma, Docker, OpenAI API, GitHub API, C++, Solidity, IPFS.

### Projects

- Required data for each project: name, role, tech stack, live link or GitHub link, architecture note, strongest 2-3 bullets, what you personally built.
- Highest-priority projects: Agent-s, De_carSharing, fin_Tech, merkleNode, Simple-Blockchain-Prototype.

### Experience

- Required data: internship/work details if available, responsibilities, technologies, outcomes.
- If no formal experience: keep projects strong and frame them as engineering work with ownership.

### Education

- Required data: degree, branch, institution, dates, graduation timeline, CGPA/percentage if useful.
- Current known timeline: B.E. AIML, 2022-08-01 to 2026-05-01, expected May 2026.

### Proof And Evidence

- Required data: demos, screenshots, GitHub READMEs, architecture notes, project case studies, open-source contributions, DSA profile if targeting MNCs.
- Missing proof themes: target job keywords, referral contact, live demo links, AI project demo, OSS proof, SQL evidence, cloud evidence, systems evidence.

## Special Notes

- For MNC resumes: emphasize CS fundamentals, DSA, clean structure, ATS safety, education clarity, backend/database basics, and interview readiness.
- For startup resumes: emphasize shipped projects, demos, ownership, fast learning, product sense, GitHub quality, and concise project proof.
- For AI companies: emphasize AI tooling, agents, developer tools, LLM API usage, automation, clear writing, and safety-aware implementation.
- For backend/platform roles: emphasize APIs, databases, reliability, testing, deployment, and system design basics.
- For systems/blockchain roles: emphasize C/C++, performance, distributed systems, cryptography/hash concepts, Solidity/IPFS where true.
- For frontend/full-stack roles: emphasize live product UI, React/Next.js, API integration, polished UX, and responsive implementation.
